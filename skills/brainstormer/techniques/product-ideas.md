# Product Ideas Skill

Generate product features, improvements, and innovation concepts.

## When to Use

- Need new product features
- Want to improve existing product
- Exploring new product lines
- Looking for product-market fit

---

## Feature Ideation Techniques

### 1. Jobs to Be Done (JTBD)

Ask: "What job does the customer hire this product to do?"

- **Functional**: Complete a task
- **Emotional**: Feel something
- **Social**: Belong somewhere

### 2. User Journey Mapping

For each stage:
- **Awareness**: How do they find us?
- **Onboarding**: How do they start?
- **Usage**: How do they use it?
- **Support**: How do they get help?
- **Expansion**: How do they use more?

### 3. Pain Point Mining

- What takes too long?
- What confuses users?
- What frustrates them?
- What's missing?
- What's broken?

### 4. Feature Categories

| Category | Examples |
|----------|----------|
| **Core** | Essential functionality |
| **Enhancement** | Make existing better |
| **Delight** | Unexpected value |
| **Retention** | Keep users engaged |
| **Monetization** | New revenue paths |
| **Efficiency** | Save time/money |

---

## New Product Ideation

### Questions to Ask

**Problem Discovery**
- What problems do target users face daily?
- What do they complain about?
- What do they wish existed?
- What takes too much time/money?

**Market Discovery**
- What's growing in this space?
- What trends could we ride?
- What's underserved?
- What's ripe for disruption?

**Capability Discovery**
- What can we uniquely do?
- What resources do we have?
- What partnerships could help?
- What tech enables new things?

## Product Concept Generator

### For Each Idea, Define:

```markdown
## Product Concept: [Name]

### One-Liner
[Clear, compelling description]

### Target User
[Specific person/persona]

### The Problem
[Pain point we're solving]

### The Solution
[How we solve it]

### Key Features (MVP)
1. [Feature 1]
2. [Feature 2]
3. [Feature 3]

### How It's Different
[Competitive edge]

### Monetization
[How we'll make money]

### Metrics to Watch
[How we measure success]
```

## Feature Prioritization Matrix

| | Low Impact | High Impact |
|---|---|---|
| **Easy** | Quick wins | Core features |
| **Hard** | Fill-in | Strategic bets |

### Prioritization Questions

1. **Will this increase retention?**
2. **Will this attract new users?**
3. **Is it technically feasible now?**
4. **What's the development cost?**
5. **Do users actually want this?**
