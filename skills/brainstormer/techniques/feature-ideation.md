# Feature Ideation Skill

Deep dive into generating and refining product features.

## When to Use

- Need specific product features
- Planning product roadmap
- Want to innovate on existing product
- Building MVP requirements

---

## Feature Categories

### 1. Core Features
- Must-haves for product to work
- The fundamental value proposition
- MVP minimum requirements

### 2. Enhanced Features
- Make existing features better
- Improved UX, performance, reliability
- Polish and optimization

### 3. Differentiating Features
- What makes us unique
- Competitive advantages
- "Wow" factors

### 4. Retention Features
- Keep users coming back
- Engagement loops
- Habit formation

### 5. Monetization Features
- New revenue streams
- Premium upgrades
- Add-ons and accessories

---

## Feature Ideation Techniques

### 1. Jobs to Be Done

Ask: "What job does the user hire our product to do?"

For each job:
- Functional jobs (complete tasks)
- Emotional jobs (feel good)
- Social jobs (look good/be part of)

### 2. User Story Mapping

Create journey:
```
[Awareness] → [Sign up] → [First use] → [Regular use] → [Value realized] → [Champion]
```

For each step:
- What works?
- What's friction?
- What's missing?

### 3. Competitor Analysis

- What features do they have?
- What are users asking for?
- What can we do better/different?

### 4. Tech-Driven

- What does new tech enable?
- What was impossible before?
- What becomes cheaper/faster?

### 5. User Request Mining

- Support tickets
- Feature requests
- App store reviews
- NPS comments
- Exit surveys

---

## Feature Discovery Questions

### By User Segment

For each user type:
- What do they struggle with?
- What takes too long?
- What confuses them?
- What do they wish existed?

### By Journey Stage

**Onboarding**
- What helps users succeed first?
- What causes drop-off?
- What delivers quick wins?

**Core Usage**
- What prevents full value?
- What features go unused?
- What do power users want?

**Expansion**
- What triggers upgrade?
- What cross-sell opportunities?
- What enterprise needs?

## Feature Prioritization

### RICE Score

```
RICE = (Reach × Impact × Confidence) / Effort

Reach = Users affected per quarter
Impact = Effect on metric (3=massive, 1=minimal)
Confidence = How sure are we (%)
Effort = Person-weeks
```

### MoSCoW

- **Must have**: Core functionality
- **Should have**: Important but not critical
- **Could have**: Desirable
- **Won't have**: Explicitly deprioritized

### Value vs Effort

| | Low Effort | High Effort |
|---|---|---|
| **High Value** | Quick wins | Strategic bets |
| **Low Value** | Fill-in | Avoid |
