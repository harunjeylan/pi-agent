# Problem Solving Skill

Structured approaches to solve any challenge.

## When to Use

- User faces a difficult problem
- Current solutions aren't working
- Need fresh perspectives
- Want systematic approach

---

## Problem Definition

### Before Solving, Clarify:

1. **What exactly is the problem?**
   - State it in one sentence
   - Remove emotion, focus on facts

2. **What caused it?**
   - Timeline of events
   - Key decisions made
   - External factors

3. **Who does it affect?**
   - Direct stakeholders
   - Indirect impacts
   - Severity levels

4. **What have we tried?**
   - Previous attempts
   - Why they failed
   - What we learned

---

## Problem Solving Frameworks

### 1. Five Whys

**How to use:**
1. State the problem
2. Ask "Why?" 5 times
3. Each answer becomes the next "why"

**Example:**
```
Problem: Launch was delayed
1. Why? Developer resources were reassigned
2. Why? Urgent bug fix needed elsewhere
3. Why? Testing wasn't thorough
4. Why? Tight deadline pressured shortcuts
5. Why? Deadline was unrealistic

Root Cause: Unrealistic deadlines
```

### 2. Fishbone Diagram

Categories to consider:
- People (who)
- Process (how)
- Technology (tools)
- Materials (what)
- Environment (where)
- Management (why)

### 3.假设检验 (Hypothesis Testing)

1. State hypothesis: "I believe X causes Y"
2. Gather evidence
3. Test assumption
4. Confirm or reject
5. Repeat

### 4. First Principles

Break down to fundamentals:
1. What assumptions am I making?
2. Can I prove each one?
3. What if these were false?
4. Build up from truth

## Solution Generation

### For Each Problem, Generate:

**Immediate Fixes**
- What stops the bleeding now?
- What's the quick patch?

**Short-Term Solutions**
- How do we fix this properly?
- What resources needed?

**Long-Term Solutions**
- How do we prevent recurrence?
- What systemic change needed?

**Opportunities**
- Any upside in this problem?
- Any lessons we can apply elsewhere?

## Problem Solving Template

```markdown
## Problem Statement
[Clear, factual description]

## Root Cause Analysis
### Timeline
- [Event 1]
- [Event 2]
- [Root cause identified]

### 5 Whys
1. Why: [Answer]
2. Why: [Answer]
3. Why: [Answer]
4. Why: [Answer]
5. Why: [Root cause]

## Solutions Considered

### Option A: [Name]
- **What**: [Description]
- **Pros**: [Benefits]
- **Cons**: [Drawbacks]
- **Cost**: [Resources needed]
- **Time**: [Duration]
- **Risk**: [Potential issues]

### Option B: [Name]
[Same structure]

## Recommended Solution
[Why this one]

## Action Plan
- [ ] Step 1
- [ ] Step 2
- [ ] Step 3

## Success Metrics
- [ ] [Metric] improved by [X%]
- [ ] [Metric] reached [Y]

## Contingency
If this doesn't work:
- [ ] Backup plan
```

## Common Problem Patterns

| Problem | Likely Cause | Common Solutions |
|---------|-------------|------------------|
| Too slow | Too many steps | Simplify |
| Too expensive | Waste/scope creep | Cut scope, optimize |
| Wrong direction | Poor planning | Reframe objectives |
| People issues | Communication/conflict | Align incentives, facilitate |
| Tech issues | Technical debt | Refactor, upgrade |
| Market issues | Misalignment | Re-validate, pivot |

## When Stuck

If no solution works:

1. **Sleep on it** - New perspectives tomorrow
2. **Ask others** - Fresh eyes
3. **Simplify** - What's the minimal fix?
4. **Pivot** - Maybe the problem is different
5. **Accept** - Some things can't be fixed
